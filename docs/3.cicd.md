# CI/CD Pipeline Guide

The WriteOps Book Template includes a robust GitHub Actions workflow to automate your book production.

## Workflow Overview

The `.github/workflows/build-pdf.yml` handles:
- **Environment Setup:** Installs Pandoc, TeX Live, and Chromium.
- **Dependency Management:** Uses `pnpm` to install `mermaid-filter`.
- **Validation:** Runs `./build.sh` to check for syntax or LaTeX errors.
- **Artifact Upload:** Uploads the generated PDF as a workflow artifact for every build.
- **Automated Releases:** Attaches the PDF to GitHub Releases for tagged versions.

## Customizing the Pipeline

You can modify the workflow to:
- Change the Pandoc version.
- Add extra LaTeX packages (using `apt-get install` in the runner).
- Trigger builds on different branches.

## Troubleshooting CI Builds

If the GitHub Action fails:
1.  Check the **"Run ./build.sh"** step in the Action logs.
2.  Look for LaTeX errors (e.g., `! LaTeX Error: File ... not found`).
3.  Ensure all images referenced in your chapters are correctly pushed to the `assets/` directory.
