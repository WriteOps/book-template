# Getting Started with the WriteOps Book Template

Welcome! This template provides a professional, automated pipeline for writing books using Markdown and generating high-quality PDFs.

## Quick Start

1.  **Use this template:** Click the "Use this template" button on GitHub to create your own repository.
2.  **Open in VS Code:** Clone your new repository and open it in VS Code.
3.  **Launch Container:** When prompted, click "Reopen in Container" (requires Docker). This ensures you have all the necessary tools installed.
4.  **Write Content:** Add your markdown files to the `chapters/` directory.
5.  **Build your PDF:**
    ```bash
    pnpm run build
    ```

## Project Structure

- `chapters/`: Your book's main content. Each subfolder should follow the `chapter-{n}` naming convention.
- `meta/`: Essential non-chapter pages:
    - `title-copyright.tex`: The professional title and copyright page (renders before TOC).
    - `preface.md`: Your book's introduction.
    - `conclusion.md`: Closing thoughts.
    - `about-author.md`: Your author bio and photo.
- `assets/`: Images, icons, and fonts used in your book.
- `templates/`: The LaTeX template (Eisvogel) used for PDF generation.
- `metadata.yml`: Configuration for your book's title, author, and styling.
- `output/`: Where your finished PDF will be generated.

## Folder & File Conventions

To ensure the build script processes your chapters in the correct order, follow these naming rules:
1.  **Chapters:** Folders must be named `chapter-1`, `chapter-2`, etc.
2.  **Files:** Inside each chapter folder, name your files `{index}.{title}.md` (e.g., `1.introduction.md`, `2.deep-dive.md`).
