# Continuous Integration and Deployment

The WriteOps Book Template comes pre-configured with GitHub Actions to ensure your book is built and released automatically.

## How it Works

The workflow file is located at `.github/workflows/build-pdf.yml`. It is triggered in two scenarios:

1.  **On Push/PR:** Every time you push code to the `main` or `master` branch, the CI pipeline runs a build check to ensure your book still compiles without errors.
2.  **On Release:** When you create a new GitHub Release with a tag starting with `v` (e.g., `v1.0.0`), the pipeline builds the final PDF and automatically attaches it to the release as an asset.

## GitHub Actions Environment

The pipeline uses the official Pandoc action and installs a minimal Chromium environment to handle Mermaid diagrams. This ensures that the PDF generated in the cloud is identical to the one you build locally or in Docker.

## Setting it Up

To enable the automated releases:
1.  Push your code to GitHub.
2.  Go to the **"Actions"** tab to see your first build running.
3.  When you're ready to publish, go to **"Releases"** -> **"Create a new release"**.
4.  Give it a tag like `v1.0.0` and publish.
5.  Wait a few minutes, and your PDF will appear in the release assets!
