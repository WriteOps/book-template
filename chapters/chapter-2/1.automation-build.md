# Automation and the Build Pipeline

One of the core strengths of this template is its automated build pipeline. You focus on the content; the system handles the layout.

## The Build Script

The `build.sh` script is the heart of the system. It performs the following steps:
1.  **Gathers Files:** It dynamically finds every markdown file in the `chapters/` directory.
2.  **Merges Content:** It creates a temporary `merged.md` file.
3.  **Fixes Paths:** It automatically adjusts image paths so they work regardless of where the source file is located.
4.  **Runs Pandoc:** It uses Pandoc and the Eisvogel LaTeX template to generate a professional-grade PDF.

## Running the Build

You have three ways to build your book:

### 1. Local Build
If you have Pandoc and LaTeX installed locally:
```bash
pnpm run build
```

### 2. Docker Build
If you don't want to mess with local installations:
```bash
docker build -t book-builder .
docker run --rm -v $(pwd):/data book-builder
```

### 3. VS Code DevContainer
The most recommended way! Just open this project in VS Code, and it will offer to reopen it in a container. Everything will be pre-installed and ready to go.
