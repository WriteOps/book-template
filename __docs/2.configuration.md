# Configuration Guide

You can customize almost every aspect of your book's appearance through `metadata.yml`.

## Book Information

Update these fields at the top of the file:
- `title`: The title of your book.
- `author`: Your name.
- `date`: Publication year.

## Styling Options

The template uses the **Eisvogel** LaTeX template. You can customize:
- `titlepage-color`: Background color of the cover page (HEX).
- `titlepage-text-color`: Text color on the cover page.
- `titlepage-rule-color`: Color of the decorative line on the cover page.
- `titlepage-background`: Path to your cover image.

## Layout Options

- `oneside`: Set to `true` to remove blank pages (recommended for digital PDFs).
- `toc`: Enable or disable the Table of Contents.
- `toc-own-page`: Ensure the TOC starts on its own page.

## Managing Assets (Images)

All images should be stored in the `assets/images/` directory.

### Linking Images
When writing in your markdown files (located in `chapters/chapter-n/`), you should link to images using a path relative to the file. This ensures you can see the images in your VS Code preview:

```markdown
![My Image](../../assets/images/my-image.png)
```

**Note:** The build script automatically converts these paths to be root-relative for the final PDF compilation.

### Remote Images (URLs)
You can use remote image URLs (HTTP/HTTPS) anywhere in your book—inside your Markdown chapters or in your `metadata.yml`.

```markdown
![Remote Image](https://raw.githubusercontent.com/WriteOps/.github/refs/heads/main/cover.png)
```

The build script automatically detects, downloads, and caches these images during the build process to ensure a smooth PDF generation.
